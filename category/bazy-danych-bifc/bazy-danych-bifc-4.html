Ćwiczenia 4 – Bazy Danych [0600-BFIF4B]

Wykonaj poniższe zadania za pomocą skryptów SQL.

  1. Załóż bazę DOMY.
  2. Utwórz tabelę DOM o kolumnach:
    • Id – liczba całkowita, autonumerowana, klucz główny (IDENTITY(1,1), PRIMARY KEY).
    • Powierzchnia – pole typu decimal(5,2) NOT NULL.
  3. Utwórz tabelę OSOBA o kolumnach:
    • Id – autonumerowane, klucz główny.
    • Imie – tekst do 50 znaków, UNICODE, wymagane.
    • Nazwisko – tekst do 50 znaków, UNICODE, wymagane.
    • Pesel – 11 znaków, UNICODE, wymagane, unikalne (UNIQUE).
  4. Utwórz tabelę UMOWA o kolumnach:
    • Id – autonumerowane, klucz główny.
    • Nr – tekst do 10 znaków, UNICODE, wymagane i unikalne.
    • Data – data, wymagane, domyślnie data bieżąca.
    • Dom_id – klucz obcy do tabeli DOM.
    • Osoba_id – klucz obcy do tabeli OSOBA.
  5. Wprowadź dane o dwóch osobach, dwóch domach i dwóch umowach. Pola ID mają być nadawane automatycznie.
    • Wykorzystaj UNION, aby wykonać INSERT wielu rekordów jednocześnie.
    • SQL UNION
    INSERT INTO [dbo].osoba (imie, nazwisko, pesel)
SELECT 'Jan', 'Kowalski', '85021012345' UNION ALL
SELECT 'Adam', 'Nowak', '76031523456'
  6. Utwórz perspektywę zwracającą listę miast, w których są zarejestrowane domy (bez powtórek).
    • CREATE VIEW [dbo].[vmiasta] AS SELECT DISTINCT(miasto) nazwa FROM dbo.dom
    • SQL DISTINCT
  7. Utwórz skrypt usuwający wszystkie tabele i perspektywę z bazy DOMY.
    • DROP TABLE [nazwaTabeli]
    • DROP VIEW [nazwaPerspektywy]
