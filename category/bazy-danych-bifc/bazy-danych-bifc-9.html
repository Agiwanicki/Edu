Ćwiczenia 9 – Bazy Danych [0600-BFIF4B]

Cel: omówienie loginów serwera, użytkowników bazy danych, uprawnień, ról oraz haszowania haseł w SQL Server.

  • Dokumentacja CREATE LOGIN
  • Dokumentacja CREATE USER
  • Dokumentacja GRANT
  • Dokumentacja REVOKE

Przykład wprowadzający

Utwórz przykładową tabelę użytkowników aplikacji, w której hasło nie będzie przechowywane jawnie, lecz jako wartość skrótu.

CREATE TABLE users (
    user_id INT IDENTITY(1,1) PRIMARY KEY,
    login NVARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(64) NOT NULL,
    email NVARCHAR(255) NOT NULL UNIQUE
);

SELECT CONVERT(VARCHAR(64), HASHBYTES('SHA2_256', 'Password1234'), 2);

INSERT INTO users (login, password_hash, email)
VALUES (
    'jan.kowalski',
    CONVERT(VARCHAR(64), HASHBYTES('SHA2_256', 'Password1234'), 2),
    'jan.kowalski@example.com'
);
  1. Utwórz dwa loginy serwera SQL Server: Login1 oraz Login2.
    • Dla pierwszego loginu ustaw hasło jawnie podczas tworzenia.
    • Dla drugiego loginu wykorzystaj hasło w postaci zaszyfrowanej lub zahaszowanej.
    • Pomocniczo sprawdź działanie funkcji PWDENCRYPT.
  2. Wyświetl listę loginów serwera dostępnych w instancji SQL Server.
    • Wykorzystaj widok systemowy sys.server_principals.
    • Wyświetl nazwę loginu oraz typ loginu.
  3. Utwórz użytkowników baz danych na podstawie wcześniej utworzonych loginów.
    • W bazie Northwind utwórz użytkownika NorthwindUser dla loginu Login1.
    • W drugiej bazie danych, np. DOMY, utwórz użytkownika OtherUser dla loginu Login2.
    • Pamiętaj o przełączeniu kontekstu bazy danych poleceniem USE.
  4. Sprawdź działanie uprawnień użytkownika NorthwindUser w bazie Northwind.
    • Przełącz kontekst wykonania na użytkownika NorthwindUser.
    • Spróbuj wykonać zapytanie odczytujące dane z tabeli Customers.
    • Następnie wróć do poprzedniego kontekstu i nadaj użytkownikowi uprawnienie SELECT.
    • Powtórz zapytanie kontrolne i porównaj wynik.
  5. Nadaj użytkownikowi NorthwindUser uprawnienia do modyfikacji danych w tabelach związanych z zamówieniami.
    • Nadaj uprawnienia UPDATE oraz INSERT do tabeli Orders.
    • Nadaj uprawnienia UPDATE oraz INSERT do tabeli Order Details.
  6. Odbierz użytkownikowi NorthwindUser wybrane uprawnienia.
    • Odbierz uprawnienie INSERT do tabeli Orders.
    • Odbierz uprawnienie INSERT do tabeli Order Details.
    • Wykorzystaj polecenie REVOKE.
  7. Zmodyfikuj właściwości loginów serwera.
    • Zmień hasło loginu Login1.
    • Wyłącz login Login2.
    • Wykorzystaj polecenie ALTER LOGIN.
  8. Przećwicz usuwanie loginu oraz zmianę powiązania użytkownika bazy danych z loginem serwera.
    • Usuń login Login2.
    • Sprawdź, co dzieje się z użytkownikiem bazy danych powiązanym z usuniętym loginem.
    • Zmień powiązanie użytkownika OtherUser tak, aby korzystał z loginu Login1.
    • Wykorzystaj polecenie ALTER USER ... WITH LOGIN.
  9. Utwórz rolę bazodanową i przypisz do niej użytkownika.
    • Utwórz rolę Salesman.
    • Nadaj roli uprawnienia SELECT oraz INSERT.
    • Dodaj użytkownika NorthwindUser do roli Salesman.
    • Na końcu wyświetl listę principalów bazy danych z widoku sys.database_principals.
