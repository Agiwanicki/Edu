Ćwiczenia 3 – Implementacja modelu bazy danych w SQL Server

Celem ćwiczenia jest implementacja zaprojektowanego wcześniej modelu bazy danych w środowisku Microsoft SQL Server Management Studio (SSMS) przy użyciu języka T-SQL.

Podczas implementacji należy wykorzystać wybrane mechanizmy dostępne w SQL Server, które wspierają:

  • strukturę danych,
  • integralność danych,
  • automatyzację operacji,
  • przetwarzanie danych.

Poniżej przedstawiono przykładowe elementy, które można zastosować w projekcie.

1. Tworzenie tabel z kluczami głównymi i obcymi

Relacyjna baza danych opiera się na relacjach między tabelami. Relacje te definiuje się za pomocą kluczy głównych (PRIMARY KEY) oraz kluczy obcych (FOREIGN KEY).

Przykład – dwie powiązane tabele

CREATE TABLE Customers (
    CustomerID INT PRIMARY KEY IDENTITY(1,1),
    Name VARCHAR(100) NOT NULL
);

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY IDENTITY(1,1),
    CustomerID INT,
    OrderDate DATE,
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);

W powyższym przykładzie:

  • CustomerID jest kluczem głównym w tabeli Customers,
  • CustomerID w tabeli Orders jest kluczem obcym,
  • oznacza to, że zamówienie musi odnosić się do istniejącego klienta.

2. Ograniczenia integralności danych

Ograniczenia (constraints) służą do zapewnienia poprawności danych przechowywanych w bazie.

a) CHECK

Ograniczenie CHECK wymusza spełnienie określonego warunku.

Przykład – cena produktu musi być dodatnia:

CREATE TABLE Products (
    ProductID INT PRIMARY KEY,
    Price DECIMAL(10,2),
    CHECK (Price > 0)
);

Można także ograniczyć wartości do określonego zbioru:

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    Status VARCHAR(20),
    CHECK (Status IN ('New','Processing','Completed','Cancelled'))
);

Przykład – wiek użytkownika musi mieścić się w określonym zakresie:

CREATE TABLE Users (
    UserID INT PRIMARY KEY,
    Age INT,
    CHECK (Age BETWEEN 18 AND 120)
);

b) UNIQUE

Ograniczenie UNIQUE zapewnia, że wartości w kolumnie są unikalne.

Przykład – adres e-mail nie może się powtarzać:

CREATE TABLE Users (
    UserID INT PRIMARY KEY,
    Email VARCHAR(100) UNIQUE
);

c) DEFAULT

Ograniczenie DEFAULT przypisuje wartość domyślną.

Przykład – automatyczne ustawienie bieżącej daty:

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    OrderDate DATE DEFAULT GETDATE()
);

Jeżeli data nie zostanie podana przy wstawianiu rekordu, zostanie użyta aktualna data systemowa.

3. Typy danych w SQL Server

Dobór odpowiednich typów danych jest istotny dla poprawnego działania bazy.

a) Typy liczbowe

Typy liczbowe służą do przechowywania wartości całkowitych, dziesiętnych oraz pieniężnych. Dobór konkretnego typu zależy od zakresu wartości i wymaganej dokładności.

Najczęściej używane typy liczbowe:

  • TINYINT – małe liczby całkowite od 0 do 255, np. liczba gwiazdek w ocenie produktu,
  • SMALLINT – mniejsze liczby całkowite, np. liczba punktów lub stan magazynowy małych zbiorów,
  • INT – podstawowy typ dla liczb całkowitych, np. identyfikatory rekordów, liczba produktów, liczba zamówień,
  • BIGINT – bardzo duże liczby całkowite; można go traktować jako odpowiednik „long” znany z innych języków programowania, np. duże identyfikatory lub liczniki zdarzeń,
  • DECIMAL(p,s) / NUMERIC(p,s) – liczby dziesiętne o określonej precyzji, np. ceny, stawki VAT, wartości rabatów,
  • FLOAT – liczby zmiennoprzecinkowe przybliżone, np. wyniki pomiarów, współczynniki lub wartości statystyczne,
  • REAL – podobny do FLOAT, ale o mniejszej precyzji,
  • MONEY – typ przeznaczony do przechowywania wartości pieniężnych,
  • SMALLMONEY – typ pieniężny o mniejszym zakresie.

Przykłady:

CustomerID INT
OrderCounter BIGINT
Price DECIMAL(10,2)
Discount FLOAT
TotalAmount MONEY

W przypadku DECIMAL(10,2):

  • pierwsza liczba oznacza łączną liczbę cyfr,
  • druga oznacza liczbę miejsc po przecinku.

b) Typy tekstowe

Typy tekstowe służą do przechowywania napisów o stałej lub zmiennej długości.

  • CHAR(n) – tekst o stałej długości; przydatny np. dla kodów pocztowych, skrótów lub stałych identyfikatorów,
  • VARCHAR(n) – tekst o zmiennej długości; dobry np. dla nazw, e-maili, adresów,
  • NCHAR(n) – tekst stałej długości w standardzie Unicode,
  • NVARCHAR(n) – tekst zmiennej długości w standardzie Unicode,
  • TEXT – starszy typ do przechowywania długiego tekstu,
  • NTEXT – starszy odpowiednik TEXT dla Unicode.

Litera N w typach NCHAR i NVARCHAR oznacza obsługę Unicode, czyli możliwość poprawnego przechowywania znaków narodowych i wielu alfabetów, np. polskich liter: ą, ć,ę, ł, ń, ó, ś, ź, ż. W praktyce NVARCHAR warto stosować wszędzie tam, gdzie tekst może zawierać znaki spoza podstawowego alfabetu ASCII.

Przykłady:

PostalCode CHAR(6)
Name VARCHAR(100)
City NVARCHAR(100)
Description TEXT

c) Typy daty i czasu

Typy tej grupy służą do przechowywania dat, czasu lub obu tych elementów jednocześnie.

  • DATE – przechowuje tylko datę,
  • DATETIME – przechowuje datę i czas,
  • DATETIME2 – nowszy typ daty i czasu o większej precyzji,
  • SMALLDATETIME – data i czas w mniejszym zakresie i z mniejszą dokładnością,
  • TIME – przechowuje wyłącznie czas, np. godzinę, minuty i sekundy.

Przykłady:

BirthDate DATE          -- 2024-10-10
OrderDate DATETIME      -- 2024-10-10 14:30:00
EventTime TIME          -- 14:30:00

Typ TIME jest przydatny np. wtedy, gdy chcemy przechować jedynie godzinę rozpoczęcia zajęć, czas wizyty albo porę dostawy bez konkretnej daty.

d) Typy logiczne

W SQL Server do przechowywania wartości logicznych używa się przede wszystkim typu BIT. Można go traktować jako odpowiednik typu bool / boolean znanego z innych języków programowania.

  • BIT – przechowuje wartości 0 lub 1, interpretowane odpowiednio jako fałsz i prawda.

Przykłady zastosowania:

  • aktywność użytkownika,
  • status zamówienia,
  • informacja, czy produkt jest dostępny,
  • informacja, czy rekord został zatwierdzony.

Przykład:

IsActive BIT
IsPaid BIT
IsArchived BIT

4. Kolumny obliczane (Computed Columns)

Kolumna obliczana to kolumna, której wartość jest automatycznie wyliczana na podstawie innych kolumn.

Przykład – obliczenie podatku:

CREATE TABLE Products (
    ProductID INT PRIMARY KEY,
    Price DECIMAL(10,2),
    Tax AS (Price * 0.23)
);

Kolumna Tax będzie automatycznie wyliczana na podstawie ceny.

Inne przykłady:

a) Wartość brutto

GrossPrice AS (Price * 1.23)

b) Pełne imię i nazwisko

FullName AS (FirstName + ' ' + LastName)

c) Wartość zamówienia

TotalValue AS (Quantity * UnitPrice)

5. Widoki (Views)

Widok jest wirtualną tabelą opartą na zapytaniu SQL. Może służyć do uproszczenia zapytań lub prezentacji danych.

a) Widok wykonujący obliczenia

CREATE VIEW ProductPrices AS
SELECT
    ProductID,
    Price,
    Price * 1.23 AS GrossPrice
FROM Products;

Widok wyświetla cenę netto oraz cenę brutto.

b) Widok łączący tekst

CREATE VIEW CustomerNames AS
SELECT
    CustomerID,
    FirstName + ' ' + LastName AS FullName
FROM Customers;

c) Widok łączący dane z wielu tabel

CREATE VIEW OrdersView AS
SELECT
    Orders.OrderID,
    Customers.Name,
    Orders.OrderDate
FROM Orders
JOIN Customers
ON Orders.CustomerID = Customers.CustomerID;

Widok łączy informacje o zamówieniu i kliencie.
