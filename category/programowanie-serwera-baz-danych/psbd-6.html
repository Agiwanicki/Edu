Ćwiczenia 6 – Triggery w SQL Server

Wykonaj poniższe przykłady w środowisku SQL Server Management Studio na bazie danych Northwind.

Celem ćwiczenia jest poznanie działania triggerów w języku T-SQL. Przykłady obejmują rejestrowanie zmian w tabeli oraz blokowanie wybranych operacji modyfikujących dane.

Czym jest trigger?

Trigger to specjalny obiekt bazy danych, który uruchamia się automatycznie w odpowiedzi na określone zdarzenie, takie jak INSERT, UPDATE lub DELETE. Triggery mogą służyć m.in. do:

  • logowania zmian w danych,
  • wymuszania dodatkowych reguł biznesowych,
  • blokowania niepożądanych operacji,
  • automatycznego wykonywania dodatkowych instrukcji po zmianie danych.

Tabele inserted i deleted

W triggerach SQL Server automatycznie udostępnia dwie specjalne, tymczasowe tabele logiczne: inserted oraz deleted.

  • inserted zawiera nowe wersje rekordów,
  • deleted zawiera stare wersje rekordów.

Dzięki nim można porównywać stan danych przed i po wykonaniu operacji modyfikującej.

Przykład 1 – tabela logów

Pierwszym krokiem jest utworzenie tabeli, która będzie przechowywała informacje o wykonanych zmianach. 

CREATE TABLE dbo.Logs (
    LogID INT IDENTITY PRIMARY KEY,
    TableName NVARCHAR(100),
    Operation NVARCHAR(10),
    OldValue NVARCHAR(MAX),
    NewValue NVARCHAR(MAX),
    ChangeDate DATETIME DEFAULT GETDATE()
);

Przykład 2 – trigger logujący zmiany w tabeli Employees

Poniższy trigger reaguje na operacje INSERT, UPDATE i DELETE wykonywane na tabeli Employees. W zależności od typu operacji zapisuje odpowiedni wpis do tabeli Logs. 

CREATE OR ALTER TRIGGER trg_Log_Employee_Changes
ON dbo.Employees
AFTER INSERT, UPDATE, DELETE
AS
BEGIN
    SET NOCOUNT ON;

    INSERT INTO dbo.Logs (TableName, Operation, OldValue, NewValue)
    SELECT 
        'Employees',
        'UPDATE',
        CONCAT('FirstName: ', d.FirstName, ', LastName: ', d.LastName),
        CONCAT('FirstName: ', i.FirstName, ', LastName: ', i.LastName)
    FROM inserted i
    JOIN deleted d ON i.EmployeeID = d.EmployeeID;

    INSERT INTO dbo.Logs (TableName, Operation, OldValue, NewValue)
    SELECT 
        'Employees',
        'INSERT',
        NULL,
        CONCAT('FirstName: ', i.FirstName, ', LastName: ', i.LastName)
    FROM inserted i
    WHERE NOT EXISTS (
        SELECT 1 FROM deleted d WHERE d.EmployeeID = i.EmployeeID
    );

    INSERT INTO dbo.Logs (TableName, Operation, OldValue, NewValue)
    SELECT 
        'Employees',
        'DELETE',
        CONCAT('FirstName: ', d.FirstName, ', LastName: ', d.LastName),
        NULL
    FROM deleted d
    WHERE NOT EXISTS (
        SELECT 1 FROM inserted i WHERE i.EmployeeID = d.EmployeeID
    );
END;

Przykład 3 – trigger blokujący zmiany w tabeli Categories

Trigger typu INSTEAD OF przechwytuje operację i wykonuje własny kod zamiast pierwotnego działania. W tym przykładzie każda próba aktualizacji lub usunięcia danych z tabeli Categories zostaje zablokowana. 

CREATE OR ALTER TRIGGER trg_Block_Categories_Modifications
ON dbo.Categories
INSTEAD OF DELETE, UPDATE
AS
BEGIN
    RAISERROR ('Zmiany i usuwanie danych w tabeli Categories są zabronione.', 16, 1);
END;

Test działania triggera logującego

Aby sprawdzić działanie triggera, wykonaj kolejno operacje dodania, modyfikacji i usunięcia przykładowego pracownika. 

INSERT INTO dbo.Employees (LastName, FirstName, Title, HireDate, City, Country)
VALUES ('Kowalski', 'Jan', 'Sales Representative', '2020-01-15', 'Warsaw', 'Poland');

UPDATE dbo.Employees
SET City = 'Krakow', Title = 'Regional Sales Manager'
WHERE FirstName = 'Jan' AND LastName = 'Kowalski';

DELETE FROM dbo.Employees
WHERE FirstName = 'Jan' AND LastName = 'Kowalski';

SELECT * FROM dbo.Logs
ORDER BY ChangeDate DESC;

Test działania triggera blokującego

Poniższe przykłady pokazują, jak przetestować blokowanie operacji UPDATE i DELETE na tabeli Categories z wykorzystaniem bloku TRY ... CATCH. 

BEGIN TRY
    UPDATE dbo.Categories
    SET CategoryName = 'Nowa nazwa testowa'
    WHERE CategoryID = 1;
END TRY
BEGIN CATCH
    PRINT 'Błąd (oczekiwany): ' + ERROR_MESSAGE();
END CATCH

BEGIN TRY
    DELETE FROM dbo.Categories
    WHERE CategoryID = 1;
END TRY
BEGIN CATCH
    PRINT 'Błąd (oczekiwany): ' + ERROR_MESSAGE();
END CATCH
